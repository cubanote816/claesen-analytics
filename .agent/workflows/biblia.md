Fuentes unificadas: PRD, Casos de Uso, Nueva Funcionalidad Prioritaria, Consejos. Objetivo: Este documento define el "Qué" y el "Por qué". Úsalo para validar funcionalidades antes de programar.

1. Visión y "Consejos de Oro"
   Filosofía: "Cero Complacencia". Respuestas basadas en hechos, no opiniones.

Estrategia de Desarrollo: "Vende el Humo". Prioriza mostrar un gráfico real funcional antes que una app perfecta.

Restricción Crítica: La base de datos Cafca es sagrada (Solo Lectura).

Idioma de Salida: Los emails/PDFs generados para usuarios finales deben ser exclusivamente en Neerlandés.

2. Funcionalidades Prioritarias (Roadmap)
   Nivel 1: Los Salvavidas (Inmediato)

Cash Flow Watchdog (Prioridad Máxima): Alerta si (Costo Real - Facturado) > 2.500€ o si pasan 30 días sin facturar.

El Auditor de Olvidos: Compara compras vs. ventas al cerrar proyecto (Detecta materiales no cobrados).

Alerta de Horas Quemadas: Monitoriza si el consumo de horas supera el % de avance físico.

Nivel 2: Asistente de Presupuestos (Pre-Mortem)

Validación antes de enviar oferta. Compara con 5 proyectos históricos similares.

Detecta riesgos (ej. "Siempre perdemos dinero en programación KNX").

Nivel 3: Estrategia y Crecimiento

Detección de Stock Obsoleto: Estrategias creativas para vender material parado > 365 días.

Upselling: Detectar clientes con instalaciones antiguas (>5 años) para ofrecer actualizaciones (LED/IoT).

3. Modelo de Datos de Negocio
   MAMO: Material, Mano de Obra, Maquinaria, Subcontratas.

Categorización: Obligatoriedad de definir Aard (Industria, Residencial, etc.) para que la IA funcione.
