# CAFCA Intelligence Hub - Technical Documentation & Memory

## 1. Project Vision

The **CAFCA Intelligence Hub** is a Business Intelligence (BI) layer built on top of the existing CAFCA ERP system. Its primary goal is to optimize project profitability through **Project DNA Sequencing**, which analyzes financial deviations between "Pre-Calculation" (Budget) and "Post-Calculation" (Actuals).

## 2. Core Architecture: "The Watchdog"

The system operates on an event-driven "Watchdog" architecture that continuously monitors projects for financial risks.

-   **CafcaWatchdogService**: The central orchestrator. It manages the auditing process (`auditProjectById`).
-   **CafcaFinancialRepository**: A dedicated repository pattern to centralize all financial calculations (Labor, Materials, Transport, Profits), ensuring "One Truth" across the application.
-   **GeminiService**: Integration with Google's Gemini AI. It acts as the "Cortex", receiving raw financial data (JSON) and generating human-readable strategic analysis (Project DNA).

## 3. Database Schema Extensions

### 3.1 Project Insights (`project_insights`)

Stores the "intelligence" generated by the AI for each project.

-   `efficiency_score` (Decimal): A 0-100 rating of project execution.
-   `critical_leak` (String): Identification of the main source of profit loss (e.g., "Labor Overrun").
-   `golden_rule` (String): A generated strategic lesson for future projects.
-   `full_dna` (JSON): Complete snapshot of financial metrics at the time of analysis.
-   `detailed_analysis` (Text): The long-form explanation generated by Gemini.

### 3.2 HR Intelligence (`project_technician_scores`)

Detailed granular tracking of worker efficiency per project.

-   `worker_ref`: Identifier for the technician.
-   `efficiency_score`: 0-100 rating based on budgeted vs. actual hours.
-   `role`: Distinction between Installation, Programming, etc.

## 4. Key Functional Modules

### 4.1 Project DNA Analysis (Forensic & Prevention)

-   **Forensic Auditor**: Analyzes finished projects to understand _why_ profit was higher or lower than expected.
-   **Prevention Auditor**: (Planned) Detects deviations in real-time during project execution.
-   **Command**: `cafca:sync-project-dna {--force}` allows manual triggering of analysis for specific or all active projects.

### 4.2 AI Budget Assistant

A Chat UI embedded in Filament that allows Project Managers to ask natural language questions about their budgets.

-   **Context Aware**: Users select a specific project context.
-   **Interface**: Built with Tailwind CSS, featuring a modern chat layout with "User" (Blue) and "Assistant" (Gray) message bubbles.

## 5. Frontend & UI (Filament V3)

### 5.1 Project Resource (`ProjectResource`)

The main dashboard for viewing project status.

-   **Status & WIP Card**: logic-heavy display of Project Status (Active, Finished, In Progress) and Capital WIP.
-   **Efficiency Bar**: A custom Blade component (`efficiency-bar.blade.php`) that visualizes the AI score with dynamic traffic-light colors (Red <50, Orange 50-79, Green >=80).
-   **Financials**: 3-column grid displaying accurate Labor, Material, and Transport costs.

### 5.2 Insight Resource (`ProjectInsightResource`)

A dedicated view for the Intelligence data.

-   **Team Performance Ranking**: A Relation Manager that renders a leaderboard of technicians sorted by efficiency for the specific project.

### 5.3 PDF Reporting

Custom PDF generation (`ProjectPdfController`) that produces a "Project Sheet" summarizing financials and Supplier costs, grouped by category.

## 6. Technical Stack

-   **FrameWork**: Laravel 10 / 11
-   **Admin Panel**: FilamentPHP V3
-   **AI**: Google Gemini Pro (via REST API)
-   **Styling**: Tailwind CSS
-   **Database**: MySQL (Cross-Database joins between `cafca` and `cafca_ii`).
